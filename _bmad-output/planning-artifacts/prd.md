# Product Requirements Document

## Extracted Product Brief Summary

### MVP features:
- Marketing site
- Client login
- Project dashboard
- Concept upload and feedback loop
- Final asset delivery

### Business rules:
- Clients can approve designs only after submitting feedback.
- Admins can only override projects within defined limits.

### Package definitions:
- Fixed-price logo packages with optional upsells for faster delivery and additional revisions.

### Workflow states:
- Draft → Review → Feedback Loop → Approval → Delivery

### Entitlement rules:
- Clients entitled to a specified number of revisions based on package purchased.

### Non-functional constraints:
- Must be maintainable by a lean team and prioritize reliability.

### Risks:
- Potential delays in design feedback could impact delivery timelines.
- Inadequate client communication might lead to misunderstandings.

### Open Questions:
- How will we handle last-minute design revisions?
- What criteria will be used for assessing client satisfaction?
